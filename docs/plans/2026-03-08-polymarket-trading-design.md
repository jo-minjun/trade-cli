# Polymarket Trading Implementation Design

**Goal:** Implement full trading capabilities for Polymarket prediction markets using `@polymarket/clob-client` SDK.

**Architecture:** Wrap `@polymarket/clob-client` SDK to conform to the existing `Exchange` interface. Two-step authentication: derive API credentials from private key, then use credentials for all trading operations.

**Tech Stack:** `@polymarket/clob-client`, `ethers@^5.8.0` (required by SDK)

---

## Scope

| Method | Implementation |
|--------|---------------|
| `placeOrder()` | CLOB API + EIP-712 signing via SDK |
| `cancelOrder()` | CLOB API order cancellation |
| `getBalance()` | USDC collateral balance query |
| `getOpenOrders()` | CLOB API open orders list |
| `getPositions()` | Conditional token balance per market |
| `getCandles()` | Throw "Not supported" error |

## Authentication Flow

1. User provides `private-key` in config
2. `prediction setup` command derives API credentials (`key`, `secret`, `passphrase`) via `createOrDeriveApiKey()`
3. Credentials are saved to config for reuse
4. Full ClobClient is initialized with both signer and credentials

## Config Changes

```yaml
prediction:
  default-via: polymarket
  polymarket:
    private-key: "0x..."        # User provides
    api-key: "..."              # Auto-generated by setup
    api-secret: "..."           # Auto-generated by setup
    api-passphrase: "..."       # Auto-generated by setup
    funder-address: "0x..."     # Polymarket profile address (optional)
```

## File Changes

### 1. Dependencies
- Add `@polymarket/clob-client`
- Add `ethers@^5.8.0`

### 2. `src/config/types.ts`
- Extend `PolymarketCredentials` with `api-key`, `api-secret`, `api-passphrase`, `funder-address`

### 3. `src/exchanges/polymarket/client.ts`
- Replace manual API calls with `ClobClient` SDK
- Constructor: initialize ClobClient with signer + optional creds
- `placeOrder()`: parse symbol (`marketId:outcome`) вЖТ find tokenId via `getMarket()` вЖТ `createAndPostOrder()`
- `cancelOrder()`: `clobClient.cancelOrder()`
- `getBalance()`: `getBalanceAllowance(COLLATERAL)` вЖТ USDC balance
- `getOpenOrders()`: `clobClient.getOpenOrders()`
- `getPositions()`: per-token `getBalanceAllowance(CONDITIONAL)`
- `getCandles()`: throw Error("Not supported for Polymarket")
- Remove duplicate type definitions (import from `types.ts`)

### 4. `src/commands/prediction.ts`
- Add `--price` option to buy/sell (0-1 probability price)
- Add `prediction setup` subcommand for API key generation

### 5. `src/index.ts`
- Pass full credentials (not just private-key) to PolymarketExchange constructor

## Symbol Convention

- CLI format: `prediction buy <market-id> <outcome> <amount> --price 0.55`
- Internal symbol: `${marketId}:${outcome}` (e.g., `0x123abc:Yes`)
- Token resolution: `getMarket(marketId)` вЖТ find token by outcome name вЖТ use `tokenId`

## Korea Access & Legal Risk

### Technical Access: Available
- South Korea is NOT in the 33 blocked countries or 4 close-only countries
- CLOB API (`clob.polymarket.com`) and Gamma API (`gamma-api.polymarket.com`) are accessible from Korean IPs
- Geoblock check endpoint: `GET https://polymarket.com/api/geoblock`

## Constraints

- `ethers` must be v5 (SDK requirement, v6 not compatible)
- Order prices are 0-1 (probability), not currency amounts
- `tickSize` and `negRisk` are per-market вАФ must fetch before placing orders
